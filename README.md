# VBD-Copilot - Slides & Demos Builder

> AI-powered presentation and demo builder for Microsoft Cloud Solution Architects and Solution Engineers

---

> [!CAUTION]
> **Run this repo inside its Dev Container** - the agents depend on Python packages, LibreOffice, and other system tools pre-installed in the container.
>
> Open in VS Code and select **Reopen in Container** when prompted, or run **Dev Containers: Reopen in Container** from the Command Palette.

---

## What This Is

CSAs and Solution Engineers spend hours assembling custom presentations and demo scripts for every customer session. Official decks are too generic, too old, or built for a different audience. Demo scripts are scattered and undocumented.

This TUI app gives you two AI conductors that produce customer-ready technical content on any topic, in minutes, calibrated to your customer's level and context:

- **Slide Conductor** - generates a complete `.pptx` presentation with speaker notes from a single prompt
- **Demo Conductor** - generates a full step-by-step demo guide with all runnable companion scripts

Both conductors research official docs first, plan with your input, build, and run automated quality review before delivering output.

> **Accelerator, not autopilot.** Always review generated content before a customer session. Treat output as a strong first draft you own and refine.

## Key SDK Concepts

### Conductor + Subagent Pattern

```python
# Conductor - orchestrates the workflow, user-routable
SLIDE_CONDUCTOR_AGENT = {
    "name": "slide-conductor",
    "infer": True,    # directly selectable
    "tools": ["task", "run_pptx_qa_checks", "bing_search", ...],
    ...
}

# Subagent - invoked by conductors via the CLI's built-in 'task' tool
RESEARCH_SUBAGENT = {
    "name": "research-subagent",
    "infer": False,   # not directly selectable, fresh context per invocation
    ...
}
```

### Subagent Invocation

Conductors delegate to subagents via the CLI's built-in `task` tool.
Each subagent runs in a **fresh context** with its own system prompt and tools -
equivalent to the legacy `#runSubagent` from GitHub Copilot Chat.

```python
# The conductor calls 'task' with agent_type + prompt:
# task(agent_type="research-subagent", prompt="Research Azure Container Apps...")
# -> subagent.started event fires
# -> subagent runs with fresh context + its own tools (bing_search, web_fetch, etc.)
# -> subagent.completed event fires
# -> result returned to conductor
```

### Custom Tools

```python
# bing_search - web search (no built-in equivalent in the CLI)
@define_tool(description="Search the web via Bing...")
def bing_search(params: BingSearchParams) -> str: ...

# run_pptx_qa_checks - programmatic layout/content QA on .pptx files
@define_tool(description="Run automated layout and content checks...")
def run_pptx_qa_checks(params: RunPptxQaChecksParams) -> str: ...
```

### PPTX Generation Skill

The `pptx-generator` skill provides python-pptx helpers via `pptx_utils.py`:

```python
from pptx_utils import *

prs = create_presentation()
create_lead_slide(prs, "Title", subtitle="Sub", level="L300")
slide = create_standard_slide(prs, "Agenda", 2, TOTAL)
add_card_grid(slide, [(MS_BLUE, "Feature 1", "Description"), ...])
save_presentation(prs, "output.pptx")
```
